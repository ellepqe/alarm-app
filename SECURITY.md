# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability, please email **ellepqe@gmail.com** instead of using the issue tracker.

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

## Security Best Practices

### Data Security
- All data is stored locally on your device
- No personal data is sent to external servers
- Room database uses encrypted storage

### Permissions
The app only requests necessary permissions:
- `SCHEDULE_EXACT_ALARM` - For setting alarms
- `VIBRATE` - For vibration feedback
- `POST_NOTIFICATIONS` - For showing notifications
- `READ_EXTERNAL_STORAGE` - For importing data
- `WRITE_EXTERNAL_STORAGE` - For exporting data

### Updates
Always keep your app updated to receive:
- Security patches
- Bug fixes
- Performance improvements

## Compliance

- ✅ GDPR compliant (no data collection)
- ✅ No advertisements
- ✅ No tracking
- ✅ Open source (transparency)
